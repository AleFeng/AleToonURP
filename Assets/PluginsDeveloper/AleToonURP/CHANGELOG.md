**AleToonURP插件功能的新增、优化与修复**

- **[阴影投射]** 开启与关闭，透明物体投射阴影的透明度阈值。

---
### **AleToon/Lit**
#### 新增

#### 优化

### **AleToon/WaterPlane**
#### 新增
- NPR水面渲染Shader。
- 示例材质球。
- 自定义ShaderGUI界面。

- **[水波 Wave]** 水面的波浪表现、强度、速度等。
- **[反射 Reflect]** 水面的反射贴图、强度、模糊、菲涅尔效果等。
- **[折射 Refract]** 水底物体的折射效果。
- **[边缘泡沫 EdgeFoam]** 水面与岸边接触的边缘位置的泡沫，设置阈值表现、距离、模糊、透明度等。

## [1.2.2] - 2026/07/18
---
### **AleToon/Lit**
#### 优化
- 精简 Shader 关键词：移除未使用的 URP 模板关键词，删除 URP14 已失效的 `_CLUSTERED_RENDERING` 与无用的 `_OUTLINE_THRESHOLDMAP_ON`，减少 Shader 变体。
- 清理死代码：未使用的 `_DETAIL` 宏分支、`hash21` 函数与冗余赋值。
#### 修复
- **[光照 Light]** 附加光照累加变量未初始化，在无附加光源时可能产生异常亮度或噪点。
- **[外描边 Outline]** 描边 Pass 在 GPU Instancing / XR 立体渲染变体下的编译问题（片元变量名错误、结构体缺少 instancing/stereo 成员）。

### **AleToon/WaterPlane**
#### 修复
- **[折射 Refract]** 改用 URP 标准的 `SampleSceneColor` 采样摄像机不透明纹理（原 `tex2D(_CameraColorTexture)` 为非规范、跨平台不可用的写法），折射效果现可正确生效。
- **[基础 Basic]** 修正 SubShader 的 `RenderPipeline` 标签为 `UniversalPipeline`。
- 修正法线贴图 `_ST` 的 CBUFFER 类型（`float3` → `float4`，恢复 SRP-Batcher 兼容），以及非规范的插值器语义（`TRXCOORD`/`TANGENT` → `TEXCOORD`）。

### **ShaderGUI（编辑器）**
#### 优化
- 材质面板每帧使用最新属性重建，避免 Undo/切换材质后属性对象过期。
- 缓存折叠面板 GUIStyle，减少编辑器 GC 分配。
- 修正 `LabelTextColor` 未使用传入样式参数的问题。

## [1.3.0] - 2026/07/21
---
### **AleToon/Lit**
#### ⚠ 破坏性变更（升级前必读）
- **新增外部依赖**：外描边改为直接引用 [OutlineSmoothNormalsGenerator](https://github.com/AleFeng/OutlineSmoothNormalsGenerator) 包内的共享HLSL，**必须先安装该包**（建议 `1.8.1`，最低 `1.7.0`），否则 `AleToonURP/Lit` 无法编译。UPM 的 `dependencies` 无法解析 git URL 包，需手动安装。
- **[外描边 Outline]** 用 **TEXCOORD 通道**存储平滑法线的网格**必须重新烘焙**。该工具 `1.7.0` 起，此通道的存储格式由「三分量原始方向」改为「两分量八面体编码」，两种格式在GPU侧无从分辨，不报错、只是描边方向整体错乱。顶点色与切线通道存储不受影响。
- **[外描边 Outline]** 新增的「存储空间」默认为**切线空间**，而 `1.2.2` 及更早烘焙的数据全部是对象空间的。请重新烘焙一次（推荐，顺带获得蒙皮支持），或把材质的「存储空间」改回 `ObjectSpace`。
#### 新增
- **[外描边 Outline]** **存储空间**（对象空间 / 切线空间）。切线空间坐标是蒙皮不变量，`SkinnedMeshRenderer` 播放动画时关节处的描边不再撕开。来源选「切线 / 顶点法线」时该项恒为对象空间，面板自动隐藏。
#### 优化
- **[外描边 Outline]** 平滑法线的解码与外扩数学不再在本插件内保留副本，改为直接引用工具包的唯一真源（`OSN_GetSmoothNormalOS` / `OSN_ApplyOutlineOffset`）。今后工具包升级存储格式时本插件自动跟进，不会再出现「工具与Shader静默不一致」。
#### 修复
- **[外描边 Outline]** TEXCOORD 各档按已废弃的三分量格式解码，导致用工具 `1.7.0+` 烘焙的数据描边方向错乱。
- **[外描边 Outline]** 完全忽略烘焙时的存储空间、恒按对象空间解码。而该工具的默认存储空间是切线空间，因此「按默认设置烘焙」得到的描边整体偏斜。

### **ShaderGUI（编辑器）**
#### 新增
- **[外描边 Outline]** 「存储空间」下拉（对象空间 / 切线空间），并在来源为「切线 / 顶点法线」时自动隐藏。

## [1.4.0] - 2026/07/22
---
### **AleToon/Lit**
#### ⚠ 破坏性变更（升级前必读）
- 以下特性此前完全不生效，本版接通后会带来**观感变化**：雾 Fog 现在会作用于 `AleToonURP/Lit` 与水面；外描边现在受附加光影响、接收主光投射阴影、透明表面的描边会随之淡出；裁剪/溶解材质现在投出与表面一致的**镂空**阴影与深度。
- 明确**不再支持烘焙光照贴图**（此前虽编译了 `LIGHTMAP_ON` 等变体，但片元从不采样 GI，等于一直不支持）。如需烘焙 GI，需自行补 `Meta` Pass。
#### 修复
- **[附加光]** URP 附加光设为 **Per-Vertex** 时 `AleToonURP/Lit` 编译失败（引用了从未定义的 `vertexLight`）。
- **[阴影投射 / 深度]** 在 **Metal / Vulkan** 后端上材质不投阴影、不写深度（两个 Pass 曾用 `only_renderers` 排除这两个后端）。
- **[外描边 Outline]** 受附加光、接收阴影、透明淡出此前均不生效（Pass 缺少对应关键字声明，相关代码为死代码）。
- **[外描边 Outline]** 与表面的裁剪透明口径不一致，导致描边镂空边缘与表面不吻合。
- **[裁剪 Clip]** 裁剪/溶解材质投出**实心**阴影、阴影面板的 `Alpha Cutoff` 滑条无效。
- **[雾 Fog]** 前向 Pass 计算了雾却从不应用；水面声明了雾关键字却无雾代码。
- **[材质捕获 MatCap]** 观察空间 UV 含相机平移项，对偏离世界原点的物体轻微错位（改为只取旋转）。
#### 新增
- **[雾 Fog]** `AleToonURP/Lit` 与水面正确接收场景雾。
- **[外描边 Outline]** 描边受附加光（点光/聚光）影响、接收主光投射阴影、透明表面描边淡出。
- **[阴影投射 / 深度]** 裁剪/溶解材质投出与表面一致的镂空阴影与深度轮廓。
#### 优化
- **[变体]** 删除 `AleToonURP/Lit` 中从不消费的 URP 关键字（SSAO、DBuffer、反射探针、光照层、光照Cookie、烘焙光照贴图、DEBUG_DISPLAY），大幅减少变体数。
- **[插值器]** 删除死插值器（阴影坐标 / 切线空间视向 / 世界视向 / SH / 光照贴图UV），顶点插值器从 10 个降到 5 个。
- **[清理]** 删除 GLES2 时代遗留的 `prefer_hlslcc gles` / `exclude_renderers d3d11_9x` pragma。
#### 说明（限制）
- 描边光照在**顶点着色器**计算，故不接收屏幕空间阴影（`_MAIN_LIGHT_SHADOWS_SCREEN`），使用贴图阴影。
- 不支持烘焙光照贴图与 DepthNormals-SSAO（无对应 Pass）。

### **AleToon/WaterPlane**
#### 修复
- **[稳定性]** 菲涅尔与边缘泡沫在极值参数（`Fresnel Factor`=10 正视水面 / `Edge Foam Blur`=0）下可能产生 NaN。
#### 优化
- **[变体]** 删除从不使用的光照贴图 / DEBUG_DISPLAY 变体；`#pragma target 2.0 → 3.0`（frag 本就需要 SM3.0）。
- **[清理]** 删除无用的主光取值与死 CBUFFER 字段 `_BumpMap_ST`。
#### 说明（限制）
- 折射、深度水色、边缘泡沫依赖 URP 的 **Depth Texture** 与 **Opaque Texture**，需在 URP Asset 中开启（编辑器面板会在缺失时提示）。
- 不支持 XR 单通道实例化（Single-Pass Instanced）渲染。

### **ShaderGUI（编辑器）**
#### 修复
- **[多选编辑]** 多选材质时，关键字 / Pass / 渲染队列 / 标签此前只作用于第一个材质，其余状态不同步。
- **[界面]** 「边缘光 / 高光 / 材质捕获」三个「遮罩贴图」折叠面板因标题重名而开合联动。
- **[性能]** 基础设置面板每帧无条件写入标签/队列/深度写入，导致材质被反复标脏；改为先比较后写。
#### 优化
- 缺失属性的报错日志按名去重（不再每帧刷屏）；下拉枚举改为按实际数值映射（兼容非连续枚举）。
